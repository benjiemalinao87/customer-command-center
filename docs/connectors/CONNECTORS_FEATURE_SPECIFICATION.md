# Connectors Feature Specification

## Overview

**Connectors** is a no-code API integration builder that allows users to design custom API integrations and make them available as actions in the Flow Builder. Similar to Zapier's "Mini-Apps" or Make.com's HTTP modules, users can configure any REST API endpoint and use it as a native action in their automation flows.

## Use Case Example

A user wants to enrich phone numbers using the AudienceAcuity API:
- Create a connector called "Phone Number Enrichment"
- Configure the API endpoint, authentication, and parameters
- The connector becomes available in Flow Builder â†’ Actions â†’ Integrations
- Users can drag and drop the "Phone Number Enrichment" action into their flows

## Field Mapping Architecture

### Overview
The field mapping system allows connector responses to automatically update contact records by mapping API response fields to either:
- **User Fields**: Built-in fields in the `contacts` table (first_name, last_name, email, phone, address, city, state, zip_code, company, etc.)
- **Custom Fields**: Workspace-specific custom fields stored in `contact_custom_fields` table

### Field Mapping Configuration Structure

Each connector action in a flow can have field mappings defined:

```json
{
  "fieldMappings": [
    {
      "sourcePath": "identity.firstName",
      "targetField": "first_name",
      "targetType": "user_field"
    },
    {
      "sourcePath": "identity.lastName",
      "targetField": "last_name",
      "targetType": "user_field"
    },
    {
      "sourcePath": "identity.address",
      "targetField": "address",
      "targetType": "user_field"
    },
    {
      "sourcePath": "data.householdIncomeMidpoint",
      "targetField": "income",
      "targetType": "custom_field"
    },
    {
      "sourcePath": "data.creditMidpoint",
      "targetField": "credit_score",
      "targetType": "custom_field"
    }
  ],
  "options": {
    "createIfNotExists": true,
    "updateExisting": true,
    "overwriteExisting": true
  }
}
```

### Field Mapping Service

```javascript
// backend/src/services/fieldMappingService.js

class FieldMappingService {
  /**
   * Apply field mappings to a contact
   */
  async applyFieldMappings(contactId, responseData, fieldMappings, options = {}) {
    const userFields = {};
    const customFields = {};

    // Extract values from response using JSONPath
    for (const mapping of fieldMappings) {
      const value = this.extractValue(responseData, mapping.sourcePath);

      if (value !== undefined && value !== null) {
        if (mapping.targetType === 'user_field') {
          userFields[mapping.targetField] = value;
        } else if (mapping.targetType === 'custom_field') {
          customFields[mapping.targetField] = value;
        }
      }
    }

    // Update contact
    if (Object.keys(userFields).length > 0) {
      await this.updateUserFields(contactId, userFields);
    }

    if (Object.keys(customFields).length > 0) {
      await this.updateCustomFields(contactId, customFields);
    }

    return { contactId, userFields, customFields };
  }

  extractValue(obj, path) {
    // Use JSONPath library or custom implementation
    return path.split('.').reduce((acc, part) => acc && acc[part], obj);
  }

  async updateUserFields(contactId, fields) {
    const { data, error } = await supabase
      .from('contacts')
      .update(fields)
      .eq('id', contactId);

    if (error) throw error;
    return data;
  }

  async updateCustomFields(contactId, fields) {
    // Upsert custom fields
    for (const [key, value] of Object.entries(fields)) {
      await supabase
        .from('contact_custom_fields')
        .upsert({
          contact_id: contactId,
          field_name: key,
          field_value: value
        }, {
          onConflict: 'contact_id,field_name'
        });
    }
  }
}
```

## Security Considerations

1. **API Key Encryption**
   - Encrypt all API keys/tokens before storing in database
   - Use environment-based encryption keys
   - Never expose raw credentials in responses

2. **Rate Limiting**
   - Implement per-connector rate limits
   - Prevent abuse of external APIs
   - Queue requests if rate limit exceeded

3. **Input Validation**
   - Sanitize all user inputs
   - Validate against input schema
   - Prevent injection attacks

4. **CORS & CSP**
   - Only allow authorized origins
   - Implement proper CORS headers

5. **Audit Logging**
   - Log all connector executions
   - Track who created/modified connectors
   - Monitor for suspicious activity
