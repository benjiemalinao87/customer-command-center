# Lead Status Filtering for Sequence Enrollment

## Overview

This feature adds lead_status validation at two points in sequence execution:
1. **Enrollment** - Only allow contacts with specific lead_status values to enter the sequence
2. **Per-Step** - Check lead_status before each step; if changed, stop the sequence

## User Stories

1. "As an agent, I want to ensure only contacts with lead status 'Lead' can enter my sequence"
2. "As an agent, I want to check if lead status is still 'Lead' before sending step 4, and stop if they converted to 'Sold'"

## Data Structures

```json
{
  "enrollmentRules": {
    "leadStatusFilter": {
      "enabled": true,
      "allowedStatuses": ["Lead", "New Lead", "Contacted"]
    }
  }
}
```

### Per-Step Lead Status Check (in message object)

```json
{
  "id": 1,
  "text": "Hello!",
  "timeValue": 5,
  "timeUnit": "Minutes",
  "leadStatusCheck": {
    "enabled": true,
    "allowedStatuses": ["Lead"]
  }
}
```

## Backward Compatibility

This implementation is **100% backward compatible** with existing workflows:

1. **Enrollment check only runs when:**
   - `enrollmentRules.leadStatusFilter.enabled === true` (strict equality)
   - `allowedStatuses` is a non-empty array

2. **Per-step check only runs when:**
   - `stepLeadStatusCheck.enabled === true` (strict equality)
   - `allowedStatuses` is a non-empty array

3. **All existing workflows continue normally** because they don't have these settings enabled.
