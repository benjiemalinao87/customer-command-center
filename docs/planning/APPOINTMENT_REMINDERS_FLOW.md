# Appointment Reminders System - How It Works

## Overview

The appointment reminder system is **NOT** a database trigger. It uses **HTTP API calls** from Cloudflare Workers to Trigger.dev to schedule reminders.

## Key Points

### âŒ What It's NOT
- **NOT a database trigger** (no PostgreSQL triggers/functions)
- **NOT a cron job** that checks appointments every minute
- **NOT polling** the database for upcoming appointments

### âœ… What It IS
- **HTTP API call** from Cloudflare Worker to Trigger.dev
- **Event-driven** - triggered at the moment of booking
- **Precisely scheduled** using Trigger.dev's `wait.until()` feature
- **Durable** - Trigger.dev persists scheduled tasks even if server restarts

---

## Code Flow

### 1. Cloudflare Worker (booking-pages-worker)

```typescript
// After creating appointment in database...
async function triggerAppointmentReminders(appointmentData) {
  await fetch('https://api.trigger.dev/api/v1/tasks/schedule-appointment-reminders/trigger', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${TRIGGER_SECRET_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      appointmentId: appointmentData.id,
      workspaceId: appointmentData.workspace_id,
      contactId: appointmentData.contact_id,
      appointmentDateTime: appointmentData.appointment_datetime,
      eventName: appointmentData.event_name
    })
  });
}
```

### 2. Trigger.dev Task (appointmentReminders.js)

```javascript
export const scheduleAppointmentReminders = task({
  id: "schedule-appointment-reminders",
  run: async (payload) => {
    const { appointmentId, appointmentDateTime } = payload;
    
    // Schedule confirmation (immediate)
    await sendAppointmentReminder.trigger({
      ...payload,
      reminderType: 'confirmation'
    });
    
    // Schedule 24hr reminder
    const reminder24hr = new Date(appointmentDateTime);
    reminder24hr.setHours(reminder24hr.getHours() - 24);
    
    await wait.until({
      date: reminder24hr,
      id: `reminder-24hr-${appointmentId}`
    });
    await sendAppointmentReminder.trigger({
      ...payload,
      reminderType: '24hr'
    });
    
    // Schedule 1hr reminder
    const reminder1hr = new Date(appointmentDateTime);
    reminder1hr.setHours(reminder1hr.getHours() - 1);
    
    await wait.until({
      date: reminder1hr,
      id: `reminder-1hr-${appointmentId}`
    });
    await sendAppointmentReminder.trigger({
      ...payload,
      reminderType: '1hr'
    });
  }
});
```

## Database Tables

### `appointment_reminder_settings`
Stores configuration for what reminders to send:
- `send_confirmation` - Send immediate confirmation?
- `send_24hr_reminder` - Send 24hr reminder?
- `send_1hr_reminder` - Send 1hr reminder?
- `sms_enabled` / `email_enabled` - Which channels?
- Message templates with merge fields

### `appointment_reminders`
Logs every reminder sent:
- `appointment_id` - Which appointment
- `reminder_type` - 'confirmation', '24hr', or '1hr'
- `channel` - 'sms' or 'email'
- `status` - 'pending', 'scheduled', 'sent', 'failed'
- `sent_at` - When it was actually sent
- `twilio_sid` / `sendgrid_message_id` - Tracking IDs
